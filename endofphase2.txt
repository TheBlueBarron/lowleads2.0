  Key design decisions:
  - Escrow is fully ledger-based: every cent movement has an escrow_transactions row — append-only, auditable
  - Stripe payouts to submitters (RELEASE → Stripe Transfer) are recorded in the ledger but the actual transfer is deferred to Phase 3 (requires submitters to onboard Stripe Connect)
  - Webhook endpoint uses a scoped child context so its Buffer content-type parser doesn't override the global JSON parser
  - All PII (phone, email, notes) encrypted with KMS envelope encryption before DB insert; decrypted only when the receiving company fetches the lead detail
